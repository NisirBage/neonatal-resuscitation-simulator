Neonatal Resuscitation Simulator

A voice-first, scenario-driven clinical training platform that guides student clinicians through the NRP protocol via real-time voice interaction, live instructor oversight, and automated performance reporting.

⭐ View on GitHub 🚀 Quick Start 📖 Documentation

Key Features

Everything an NRP training lab needs — in the browser.

🎙️

Voice-First Interaction

The simulator speaks each NRP prompt aloud. Students answer verbally — "yes" or "no" — using the browser's Web Speech API. Manual buttons provide a fallback.

🔄

Finite State Machine Engine

All clinical logic lives in a JSON scenario file. The FSM enforces valid transitions, records self-loops (warm/dry, position airway), and never skips a protocol step.

📡

Live Instructor Dashboard

Real-time FSM state display, override panel, manual timer controls, and a live event log — all pushed via WebSocket. No polling required on the instructor side.

⏱️

Birth Timer & Ventilation Countdown

An elapsed birth clock starts at session creation. A ventilation countdown bar appears during time-critical steps, with per-minute voice announcements.

📊

Session Replay & Metrics

After the session ends, step through every event with playback controls. A training score and metric breakdown are computed in a single O(N) pass over the event log.

📄

Four Export Formats

PDF performance report, raw event CSV, clinical timeline CSV (clinical language), and colour-coded clinical XLSX — all generated on demand from the immutable event log.

💾

Persistent Sessions

Every state transition is written to SQLite (PostgreSQL-ready). Sessions survive backend restarts — the FSM is fully reconstructed from the persisted event history.

🔁

Auto-Reconnecting WebSocket

Exponential backoff (1 s → 30 s) keeps the student and instructor connected through network blips. State re-syncs automatically on reconnect.

Design Patterns

Production engineering patterns applied throughout the codebase.

🔀

Finite State Machine

All scenario logic is data, not code. The FSM engine reads scenarios/*.json and enforces valid transitions — adding a new scenario requires zero backend changes.

📝

Event Sourcing

Every state change is an immutable SimulationEvent appended to the database. Replay, metrics, and all four export formats are derived from this single source of truth.

📣

Pub/Sub EventBus

The FSM publishes events to an in-process EventBus. WebSocket broadcasting, timer scheduling, and persistence each subscribe independently — fully decoupled.

🗃️

Repository Pattern

A dedicated service layer isolates all database access. Swapping SQLite for PostgreSQL requires changing one connection string — no application logic changes.
